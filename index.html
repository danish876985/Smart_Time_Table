Setting Up Google Drive Sync
warning Google requires an https:// origin. Running the file locally (file:///) will trigger a Error 400: invalid_request. You must deploy first.
1
Deploy the app first. Upload this HTML file to a free static host ‚Äî GitHub Pages or Netlify are both free and take under 2 minutes. You'll get a secure https:// URL.
2
Google Cloud Console. Go to console.cloud.google.com, create a project, go to APIs & Services, and enable the Google Drive API.
3
Create credentials. Go to Credentials ‚Üí Create Credentials ‚Üí OAuth 2.0 Client ID. Set Application type to Web application.
4
Add your deployed URL. Under both "Authorized JavaScript origins" and "Authorized redirect URIs", paste your exact https://your-app.github.io URL ‚Äî no trailing slash.
5
Configure the app. Copy your Client ID (ends in .apps.googleusercontent.com), then open the SmartDay Welcome screen (or ‚öôÔ∏è Settings) and paste it into the Client ID field. Hit Save ‚Äî then tap Connect Drive.

Security & Privacy
verified_user
Your API key lives only in two places: browser memory (cleared on tab close) and your own private Drive file (smartday_sync.json). No SmartDay server ever touches it.

Drive scope: SmartDay only requests drive.file ‚Äî it can only see files it created.

Token lifecycle: The Drive token stays active for the lifetime of your browser tab. It only expires if the Google API returns a 401, at which point you're prompted to reconnect.
